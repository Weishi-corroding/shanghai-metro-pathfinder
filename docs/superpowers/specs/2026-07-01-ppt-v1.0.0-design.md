### 3.4 第 4 页 · 最短时间算法（Dijkstra + Yen K 短）

**布局**：上下两块，各 300px 高

**页标题**：最短时间：Dijkstra + Yen's K-Shortest Paths

**上半 · Dijkstra 核心**（y=140–420）：

*左侧伪代码块*（x=72, 宽 620，底色 `#0F2340`, 边框 `#1E3A5F`）：
```cpp
priority_queue<Entry, vector<Entry>, greater<>> pq;
dist[src] = 0;
pq.push({0, src, ""});
while (!pq.empty()) {
    auto [t, u, prev] = pq.top(); pq.pop();
    if (u == dst) break;
    for (const Edge& e : graph.neighbors(u, mgr)) {
        int nt = t + e.time;
        if (nt < dist[e.to]) {
            dist[e.to] = nt;
            prev_map[e.to] = {u, e.line};
            pq.push({nt, e.to, e.line});
        }
    }
}
```

*右侧要点*（x=720, 宽 490）：
- **最小堆**：`priority_queue` + `std::greater<>`
- **权重比较**：`TimeWeight` 自然序 `operator<`
- **闭合站过滤**：`Graph::neighbors()` 内联过滤
- **前驱记录**：`PredecessorInfo` 按值存储（Edge 是临时值）
- 底部小字：⚠ 反转 `operator<` 曾导致路径次优 bug（已修复）

**下半 · Yen K 短路径扩展**（y=440–680）：

三步流程横向 3 卡片：
```
[1] Dijkstra 首条  →  [2] Spur 生成候选  →  [3] 去重 & 排序取 K
    最优路径             对首条每个节点        set<vector> 哈希去重
                        分裂重跑 Dijkstra     按总时间排序
                        临时禁用节点/边
```

底部一句：**C++ 侧模板参数化 `yen_k<TimeWeight> / yen_k<TransferWeight>` — 一份代码复用两次**

### 3.5 第 5 页 · 最少换乘算法（元组权重）

**布局**：上下两块

**页标题**：最少换乘：元组权重与换乘语义

**上半 · 元组权重设计**（y=140–350）：

*左侧概念卡*（x=72, 宽 520）：
```cpp
struct TransferWeight {
    int transfers;      // 主键
    int total_time;     // 次键

    bool operator<(const TransferWeight& o) const {
        if (transfers != o.transfers)
            return transfers < o.transfers;
        return total_time < o.total_time;   // 字典序
    }
};
```

*右侧说明*（x=650, 宽 560）：
- **字典序比较**：先比换乘次数，相等时比总时间
- **Dijkstra 通用性**：只要 `operator<` 满足传递性 → 直接复用
- **同一 `dijkstra<W>` 模板**：`TimeWeight` 用于第 4 页，`TransferWeight` 用于本页

**下半 · 换乘计数语义（v1.0.0 核心修复）**（y=370–680）：

页面强调标题（金色）：**换乘 = 骑行线路之间的切换**（此定义排除了起终点的平台移动）

三条规则卡片横排：

| 规则 | 说明 |
|---|---|
| **① 起点非换乘** | 多线站起点选择首条线不算换乘（`line_trace` 不播种起点线名）|
| **② 终点非换乘** | 走到目的站台的最后一段换乘边不算换乘（`i+1 == size` 特判）|
| **③ 消重复标记** | `format()` 中连续 `--[换乘]--` 合并为一个（`prev_emitted_xfer`）|

底部示例（22px, 深蓝底卡片）：
- `莘庄 → 陆家嘴 = 1 换乘`（乘 1 号线，人民广场换 2 号线）

### 4.2 generate_slides.py 骨架

- Python 3，只用标准库 + `python-pptx`（若可用；否则纯 SVG 输出让用户手动导入 PowerPoint）
- 每页一个 `def slide_NN_xxx() -> str` 函数返回 SVG 字符串
- 全局常量集中：`CANVAS_W`, `CANVAS_H`, `COLORS = {...}`, `LINE_COLORS = {...}`, `FONT_FAMILY`
- 辅助函数：`text(x, y, content, size=24, color="#FFF", weight=400, anchor="start")`, `rect(x, y, w, h, fill, rx=0, stroke=None)`, `line(x1, y1, x2, y2, stroke, width=8)`, `circle(cx, cy, r, fill, stroke=None)`, `card(x, y, w, h, title, value, color="#FFD54F")`
- `main()` 遍历 8 页 → 写入 `svg_output/` → 调用 `svg_to_pptx()` 合成 `.pptx`
- 若 `python-pptx` 不可用：仅生成 SVG，脚本末尾提示用户手工用 PowerPoint 打开各 SVG 或用外部工具批量转换

### 4.3 SVG → PPTX 策略

**方案 A（首选）**：`python-pptx` + `svglib` + `reportlab`
- 每页 SVG → 转 PNG（scale=2 保证 2560×1440 高清）→ 作为整张 slide 背景嵌入 PPT
- 缺点：文字无法编辑，但视觉一致性最高
- 优点：跨平台可靠

**方案 B（备选）**：仅生成 SVG，让用户在 PowerPoint 手动"插入 → 图片"
- 无 Python 依赖
- 用户需要手动 8 次
- 每张 SVG 用 Inkscape 或浏览器另存 PNG 也可

**选择**：脚本默认走方案 A（检测到 `python-pptx` 装了就用），否则回退方案 B 输出说明文字。
